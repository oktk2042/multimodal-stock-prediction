# 技術スキルマップ — 修士研究から実証される専門性

本ドキュメントは、修士研究（Multi-Modal Gated Transformer）のコードベースから客観的に実証される技術領域とレベル感を整理したものである。

## 評価基準

| レベル | 定義 | 目安 |
|-------|------|------|
| Lv.1 入門 | 基本概念を理解し、チュートリアルを再現できる | 授業で触れた程度 |
| Lv.2 初級 | 簡単なタスクを単独で遂行できる | 個人プロジェクトで使用 |
| Lv.3 中級 | 実務的なタスクを独力で設計・実装できる | 実務1-2年相当 |
| Lv.4 上級 | 複雑な問題に対して設計判断を下し、最適な手法を選択できる | 実務3-5年相当 |
| Lv.5 専門家 | 当該分野で革新的な手法を提案し、他者を指導できる | 業界で認知される水準 |

## 1. 深層学習・機械学習

### 全体レベル: Lv.4（上級）

| サブスキル | レベル | 根拠（コードからの実証） |
|-----------|-------|----------------------|
| **PyTorchモデル設計** | Lv.4 | 6種のアーキテクチャ（Transformer, LSTM, CNN, 線形モデル）をnn.Moduleで自作実装。マルチモーダル融合、Gated Cross-Attention等の独自構造を設計 |
| **Attention機構の実装** | Lv.4 | Scaled Dot-Product Attention、Multi-Head Attention、Cross-Attentionをフルスクラッチ実装。Pre-Norm/Post-Normの両方に対応 |
| **GPU最適化** | Lv.4 | Mixed Precision Training（FP16 + GradScaler）、GPU常駐データローダー、デバイス間テンソル移動の適切な管理 |
| **カスタム損失関数** | Lv.4 | DirectionalMSELoss（方向ペナルティ付きMSE）を設計。モデル特性に応じた損失関数の使い分け（SmoothL1, L1, MSE, DirectionalMSE） |
| **ハイパーパラメータ最適化** | Lv.3 | Optunaで30-50 trialsの自動探索。モデル別のサーチスペース設計、Pruning、Early Stopping統合 |
| **転移学習・ファインチューニング** | Lv.3 | HuggingFace Transformersを用いたBERTのドメイン特化ファインチューニング（chABSAデータセット） |
| **勾配ブースティング** | Lv.3 | LightGBMのOptuna最適化（9パラメータ探索空間、Early Stopping） |
| **アブレーション研究** | Lv.4 | モデルアーキテクチャにablation_modeを組み込み、コンポーネント単位の寄与度を体系的に検証 |
| **特徴量重要度分析** | Lv.3 | Permutation Importance、LightGBMのGain-based Importance |

**強みの要約:** Transformerアーキテクチャの理解と実装力が特に高い。既存モデル（PatchTST, iTransformer, DLinear）の忠実な再実装に加え、独自のGated Fusion機構を提案・実装できている。

**成長余地:** 分散学習（DDP/FSDP）、モデル圧縮（量子化・蒸留）、大規模モデル学習の経験はまだない。

## 独自の強み（他の候補者との差別化ポイント）

### 1. マルチモーダルAIの設計・実装経験
テキスト（NLP）と数値時系列（Transformer）を**ゲーティング機構で動的に融合**するアーキテクチャを自ら設計・実装した経験。単一モダリティの研究者が多い中、異種データの統合を扱えることは希少。

### 2. End-to-Endのデータパイプライン構築力
データ収集（5種API）→ 前処理 → NLP → 特徴量統合 → モデル学習 → 評価 → バックテストまでの**全工程を60+スクリプトで一人で構築**。「モデルだけ作れる」ではなく「データからプロダクトまで作れる」人材。

### 3. 日本語金融ドメインの専門知識
EDINET/XBRL/J-Quants/TDnetといった日本市場固有のデータソースに精通。FinBERTの日本語ファインチューニング、XBRL日本基準とIFRSの差異を理解した上でのハイブリッド抽出等、ドメイン知識が実装に反映されている。

### 4. 解釈可能なAIへの志向
ブラックボックスで終わらせず、Gate値の可視化・ケーススタディ分析・セクター別評価等、「なぜそう予測したか」を説明できるモデル設計を一貫して行っている。
