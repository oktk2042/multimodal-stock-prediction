# Multi-Modal Gated Transformer 改善設計ノート

本ドキュメントは、修士論文で構築した Multi-Modal Gated Transformer の改善方針を整理したものである。
今後の研究・開発の方向性を検討する際の参照用として記録する。

---

## 現状の課題サマリ

| 項目 | 現状 | 課題 |
|------|------|------|
| 方向正解率（Accuracy） | 54.86%（全モデル中最高） | SOTAを上回るが、50%からの改善幅は約5pp |
| リターン予測精度（R²） | -0.002（全モデルで負値） | **方向は当たるが、変動幅の予測力が弱い** |
| 検証方法 | 単一期間ホールドアウト（2024年） | 統計的有意性検定・ウォークフォワード未実施 |
| Gate機構 | Sigmoidスカラー制御 | 不確実性を表現できない |
| テキスト特徴 | Sentiment中心 | ニュースの意味構造を十分に活用していない |

## 0. 予測精度（R²）の改善 — 最優先課題

### 問題の本質

現在のモデルは「上がるか下がるか」の方向予測には優れるが（Accuracy 54.86%）、「どれだけ動くか」の大きさ予測では全モデルがR²負値となっている。これは金融時系列予測の本質的な難しさであるが、以下のアプローチで改善の余地がある。

### 改善アプローチ

#### A. ターゲット変数の再設計

- **現状**: 5日先リターン（連続値）を直接回帰
- **改善案1**: リターンを分位点で離散化（例: 5クラス分類）し、分類問題として定式化
  - 上位20% / やや上昇 / 横ばい / やや下落 / 下位20%
  - クラス確率から期待リターンを算出
- **改善案2**: 方向（2値分類）と大きさ（回帰）を分離するTwo-Head構造
  - Head 1: 上昇/下落の分類（CrossEntropyLoss）
  - Head 2: 変動幅の回帰（MSELoss）
  - 推論時に結合: 予測リターン = sign(Head1) × |Head2|
- **改善案3**: Quantile Regression（分位点回帰）で予測分布を出力
  - 中央値 + 上下の信頼区間を同時予測
  - Pinball Lossの採用

#### B. 損失関数の改良

- **現状**: DirectionalMSELoss（方向ペナルティ付きMSE）
- **改善案**: 複合損失関数の導入
  ```
  L = α·MSE + β·DirectionalLoss + γ·RankingLoss
  ```
  - **RankingLoss**: 銘柄間の相対的な順序を学習（「A > B > C」の順序が正しければOK）
  - 株式投資では絶対値より相対ランキングが重要（上位N銘柄をロング）
  - ListwiseまたはPairwiseのLearning-to-Rank手法を応用

#### C. アンサンブル戦略

- 方向予測が得意なモデル（提案手法）と大きさ予測が得意なモデル（Ridge/LightGBM）を組み合わせ
- Stacking: 各モデルの予測値をメタモデルへ入力

**インパクト: ★★★★★（実用性に直結）**

## 1. 問題設定レベルの改善

### 1.1 Gatingの意味づけ強化

**現状:** Gating = ノイズ抑制・情報通過制御

**改善方針:** Gate値はどの市場状態を反映しているのかを定量的に検証する

**具体化:**
- Gate値とイベント（決算・政策・事故）との相関分析
- Gate値とボラティリティの関係分析（VIX, 実現ボラティリティ）
- Gate値を潜在状態モデル（Hidden Markov Model等）として扱い、市場レジームとの対応を検証

**研究価値:** Gatingを"補助機構"から"市場状態推定器"へ昇格させることで、提案手法の理論的根拠が強化される

**インパクト: ★★★★★**

---

### 1.2 モデルの適用領域明確化

**現状:** 全期間一括評価

**改善方針:** モデルが「いつ強く、いつ弱いか」を定義する

**具体化:**
- 高ボラティリティ期 vs 低ボラティリティ期での精度比較
- 金利上昇局面 vs 横ばい局面での挙動分析
- コロナショック・ウクライナ危機等の危機期特化分析
- ウォークフォワード検証による時間安定性の確認

**研究価値:** 実運用可能性が飛躍的に向上し、「どの環境で使えるか」を明示できる

**インパクト: ★★★★☆**

## 2. モデル構造レベルの改善

### 2.1 Gatingを確率モデル化

**現状:** Sigmoidでスカラー制御（決定論的）

**改善方針:** Gateを確率分布としてモデリングし、不確実性を出力する

**具体化:**
- **Bayesian Gate**: Gate重みに事後分布を仮定し、MCドロップアウトで近似
- **Concrete Distribution**: Gumbel-Softmaxによる離散的なON/OFFゲーティング
- **Variational Gate**: VAEの潜在変数としてGateを扱い、KLダイバージェンスで正則化

**研究価値:** 金融×不確実性推定という強い方向性。予測の「自信度」を出力でき、ポジションサイジングに直接活用可能

**インパクト: ★★★★★**

---

### 2.2 Cross-Attentionの解釈性強化

**現状:** Gate事例分析あり（定性的）

**改善方針:** AttentionとGateの同時可視化により情報フローを定量的に可視化する

**具体化:**
- Attention重み × Gate値のヒートマップ（時刻×特徴量の2次元）
- 時系列変化の動画・アニメーション可視化
- SHAP値やIntegrated Gradientsとの比較検証

**研究価値:** 解釈可能AI（XAI）としての価値が向上。金融規制（説明責任）への適合性も主張できる

**インパクト: ★★★★☆**

---

### 2.3 時間スケールの多段化

**現状:** 60日の固定ウィンドウで5日先を予測

**改善方針:** 複数の時間スケールを同時にモデリングする

**具体化:**
- **Multi-Scale Attention**: 短期（5日）・中期（20日）・長期（60日）の3スケールで特徴抽出し、スケール間Attentionで統合
- **Hierarchical Transformer**: 日次→週次→月次の階層的エンコーディング
- 予測ホライズンの可変化（1日/5日/20日を同時予測するMulti-Task Learning）

**研究価値:** 投資家の時間軸の違いに対応でき、実用範囲が広がる

**インパクト: ★★★★☆**

## 3. LLM活用の深化

### 3.1 LLMを弱教師として利用

**現状:** LLMは要約・重要度判定の前処理ツール

**改善方針:** LLMに「価格影響確率」を推定させ、Gate学習の教師信号にする

**具体化:**
- LLMにニュースの市場インパクト確率（0-1）を推定させる
- LLMスコア vs 実際のGate値を比較し、一致度を分析
- LLMスコアをソフトラベルとした知識蒸留（Knowledge Distillation）
- 自己学習ループ: Gate値 → LLMプロンプト改善 → より良いスコア → Gate改善

**研究価値:** LLM×金融モデルの協調学習という先端的テーマ

**インパクト: ★★★★★**

---

### 3.2 感情以外の意味特徴追加

**現状:** Sentiment（正/負/中立）中心

**改善方針:** ニュースの不確実性・新規性・影響範囲を特徴量化する

**具体化:**
- **Uncertainty score**: 「〜の可能性」「検討中」等の表現から不確実性を定量化
- **Novelty detection**: 過去のニュースとの類似度を計算し、初出情報ほど高スコア
- **Entity impact scope**: 影響範囲（個社/セクター/市場全体）の分類
- **Temporal relevance**: 情報の鮮度（既報の繰り返しは減衰）

**研究価値:** テキスト利用の深化。Sentimentだけでは捉えられない情報を活用

**インパクト: ★★★★☆**

## 4. 評価設計の改善

### 4.1 統計的有意性の検証

**現状:** 単一のテスト期間での点推定

**改善方針:** 結果の頑健性を統計的に示す

**具体化:**
- **ブートストラップ検定**: テストデータをリサンプリングし、Accuracyの信頼区間を算出
- **Diebold-Mariano検定**: モデル間の予測精度差の統計的有意性を検証
- **ウォークフォワード検証**: 学習期間をスライドさせ、複数期間での安定性を確認
- **クロスバリデーション**: TimeSeriesSplitによる時系列対応のCV

**研究価値:** 国際論文投稿時に必須。「たまたま良かった」という批判を防ぐ

**インパクト: ★★★★★**

---

### 4.2 Calibration評価の追加

**現状:** Accuracy（方向正解率）中心の評価

**改善方針:** 予測確信度と実際の成功率の整合性を検証する

**具体化:**
- **Brier Score**: 確率予測の精度を測定
- **Reliability Diagram**: 予測確率 vs 実際の成功率のプロット
- **Expected Calibration Error (ECE)**: キャリブレーション誤差の定量化
- Gate値をConfidence指標として活用（Gate値が高い予測は信頼できるか？）

**研究価値:** 実運用では「この予測はどれくらい信頼できるか」が最重要。Calibrationが良ければポジションサイジングに直結

**インパクト: ★★★★★**

---

### 4.3 失敗事例の体系化

**現状:** 成功事例（Gate Open/Close）の可視化が中心

**改善方針:** Gateが誤作動したケースを分類し、原因を体系化する

**具体化:**
- **False Open**: ノイズなのにGateが開いて精度低下したケース
- **False Close**: 重要ニュースなのにGateが閉じて見逃したケース
- 原因分類: データ不足 / FinBERTの誤判定 / 未知イベントタイプ / 市場の非合理的反応
- 失敗パターンに基づくモデル改善の方向性を導出

**研究価値:** 国際論文では「limitation analysis」として高く評価される

**インパクト: ★★★★☆**

## 5. 汎用化・発展方向

### 5.1 金融以外への転用可能性

**現状:** 日本株市場に特化

**改善方針:** 「情報がスパースで、時々重要になる」問題への一般化を示す

**具体化:**
- **医療ログ分析**: 患者の定常データ（バイタル）+ 非定常イベント（検査結果、症状報告）
- **異常検知**: センサーデータ + アラートログの動的統合
- **セキュリティログ**: ネットワークトラフィック + 脅威インテリジェンス
- **製造業**: 生産データ + 品質レポート・不良報告

**研究価値:** モデルを「ドメイン特化ツール」から「設計パターン・理論」へ昇格させる

**インパクト: ★★★★★**

---

### 5.2 情報理論的解析

**改善方針:** Gateによる情報量変化をMutual Informationで評価する

**具体化:**
- Gate前後の特徴量とターゲット間の相互情報量を計算
- Gateが情報ボトルネックとして機能していることを理論的に示す
- Information Bottleneck理論との接続

**研究価値:** 理論的裏付けが得られれば、博士課程レベルの成果に発展可能

**インパクト: ★★★★★（博士級）**

## 改善の優先度マトリクス

| 優先度 | 改善項目 | 理由 |
|-------|--------|------|
| **最優先** | R²改善（ターゲット再設計・損失関数改良） | 実用性に直結。方向+大きさの両立で投資成果が劇的に向上 |
| **最優先** | 統計的有意性の検証 | 論文投稿・実務適用の前提条件 |
| **高** | Calibration評価 | 実運用時のポジションサイジングに必須 |
| **高** | Gatingの意味づけ強化 | 学術的な差別化ポイント |
| **高** | Gatingの確率モデル化 | 不確実性推定は金融AIの最重要課題 |
| **中** | LLMを弱教師として利用 | 先端的だが実装コストが高い |
| **中** | 失敗事例の体系化 | 論文の質を大幅に向上 |
| **中** | 時間スケールの多段化 | 精度改善の可能性が高い |
| **低** | 汎用化・他ドメイン転用 | 長期的なビジョン。博士課程向け |
| **低** | 情報理論的解析 | 理論研究。博士課程向け |
